# Contributing

Blue-Print への貢献を歓迎します。会計・税務データを壊さないことを最優先にします。

## 開発手順

1. Issue または要件 ID を変更内容へ紐付けます。
2. 仕様変更は対応する ADR、要件、制限を先に更新します。
3. `swift test` を実行します。
4. DB 変更には前版サンプルからのマイグレーションテストを追加します。
5. 税務計算には対象年度、一次資料 URL、確認日、期待値を含めます。

## コーディング原則

- 金額へ `Double` を使わない。
- 転記済み仕訳や証憑原本を物理削除・上書きしない。
- UI から DB を直接変更しない。
- 氏名、住所、口座、証憑本文をログへ出力しない。
- エラー文は原因、影響、修正方法を示す。

## Pull Request

PR には要件 ID、変更理由、正常系・境界値・異常系テスト、マイグレーション影響、画面変更時の VoiceOver／キーボード確認結果を記載してください。
