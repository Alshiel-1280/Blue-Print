# Security Policy

## 対応範囲

最新の安定版と、データ移行元としてサポート中の直前バージョンを対象にします。開発版では、影響する DB スキーマ版と再現条件を明記してください。

## 報告方法

GitHub の Security Advisory の非公開報告機能を使用してください。利用できない場合は、公開 Issue には詳細を書かず、メンテナーへ非公開連絡手段の案内を依頼してください。

報告には次を含めてください。

- 影響するアプリ版・DB スキーマ版
- 再現手順と期待動作
- データ破損・漏えい・誤計算の可能性
- 可能なら匿名化した最小再現データ

氏名、住所、マイナンバー、口座番号、実在する証憑、バックアップ鍵、パスフレーズを送らないでください。
