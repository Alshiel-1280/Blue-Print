# 個人事業主向け macOS 青色申告アプリ 実装計画

- 計画バージョン: v1.0
- 作成日: 2026-07-21
- 参照: [要件定義書.md](./要件定義書.md)

## 1. 計画の目的

本書は、要件定義書を実装可能なマイルストーンと作業単位へ分解する。実装は会計の正確性、データ耐久性、証憑保存、申告出力の順に積み上げ、各バージョンで動作する成果物と回帰テストを残す。

日付ありきで機能を詰め込まず、各バージョンの品質ゲートを満たした時点で次へ進む。

## 2. 実装方針

### 2.1 優先順位

1. データを壊さないこと
2. 貸借と帳簿が一致すること
3. 原本と変更履歴を追跡できること
4. 日常記帳が完了できること
5. 決算が完了できること
6. e-Tax WEB 版へ正しいデータを渡せること
7. 操作効率と自動化を高めること
8. iPhone 等へ拡張すること

### 2.2 基本アーキテクチャ

詳細は v0.1 の ADR で確定するが、次を初期方針とする。

- UI: SwiftUI を中心としたネイティブ macOS アプリ
- 言語: Swift
- 永続化: SQLite と明示的なスキーママイグレーション
- 金額: 整数円または明示的な十進数型
- OCR: Apple のオンデバイス認識機能を第一候補とする
- PDF: macOS の PDF 機能を利用し、帳票生成層を UI から分離する
- 構成: 会計ドメイン、永続化、証憑、取込、請求、決算、税務、e-Tax、UI を分割する
- 正本: Mac のローカル DB と証憑ストア
- 外部通信: 通常の記帳・OCRでは不要

### 2.3 想定モジュール

| モジュール | 責務 |
| --- | --- |
| App | macOS エントリポイント、ナビゲーション、依存注入 |
| Domain | 金額、仕訳、勘定科目、事業者、年度、不変条件 |
| Persistence | SQLite、リポジトリ、トランザクション、マイグレーション |
| Audit | 監査イベント、取消・訂正、年度ロック |
| Imports | CSV、弥生、銀行・カード、外部 XML の取込 |
| Documents | 証憑ストア、ハッシュ、OCR、検索、電子／紙区分 |
| Invoicing | 顧客、請求書、売掛、入金、PDF |
| Payables | 外注先、受領請求書、未払、支払、源泉設定 |
| Closing | 固定資産、減価償却、家事按分、棚卸、決算整理 |
| Reporting | 帳簿、試算表、損益計算書、貸借対照表、出力 |
| Filing | 事業外所得、青色申告、申告チェック |
| ETax | 年度別フィールド、スキーマ検証、`.xtx` 生成 |
| Backup | バックアップ、暗号化、復元、整合性診断 |
| SharedCapture | 将来の Mac・iPhone 共通証憑モデルと転送契約 |

## 4. 共通の開発ルール

### 4.1 ブランチとリリース

- 変更は要件 ID または課題 ID と紐付ける。
- 機能追加は `vX.Y` の対象マイルストーンへ割り当てる。
- パッチは影響範囲を限定し、再現テストを先に追加する。
- DB 変更は必ずマイグレーションと旧版データのテストを含める。
- 税務計算変更は対象年度、根拠、期待値を含むテストを必須とする。
- リリースタグとアプリ表示バージョンを一致させる。

### 4.2 Definition of Done

各作業は以下を満たして完了とする。

- [ ] 対応する要件 ID が記載されている
- [ ] 正常系・境界値・異常系のテストがある
- [ ] データ変更がある場合はマイグレーションテストがある
- [ ] エラー時にデータが部分反映されない
- [ ] キーボードと VoiceOver の基本動作を確認した
- [ ] 利用者向け文言とエラーが具体的である
- [ ] ログへ個人・税務データを不用意に出力しない
- [ ] 仕様・制限・根拠資料が更新されている
- [ ] 対象バージョンの回帰テストが通る

### 4.3 各リリースの品質ゲート

- [ ] ユニットテスト成功
- [ ] DB マイグレーションテスト成功
- [ ] インポート・エクスポート往復テスト成功
- [ ] バックアップ・復元テスト成功
- [ ] サンプル帳簿の貸借・決算書一致
- [ ] 既存年度のゴールデンテスト結果が不変
- [ ] 依存ライセンス・脆弱性確認
- [ ] リリースノート作成
- [ ] 既知の制限を明記

## 7. 税制・e-Tax 更新手順

新しい申告年度へ対応するときは、以下の順に進める。

1. 国税庁・財務省の一次資料を収集し、確認日を記録する。
2. 変更点を税率、控除、保存、帳票、e-Tax の区分で整理する。
3. 既存年度ルールを複製せず、新しい版として追加する。
4. 根拠付きの正解ケースを先に作る。
5. 計算・帳票・`.xtx` を実装する。
6. 旧年度の全ゴールデンテストを実行する。
7. 対象年度の e-Tax WEB 版または公開検証手段で確認する。
8. 対応範囲、未対応、暫定仕様をリリースノートへ記載する。
9. 機能追加として `vX.Y` を公開する。
10. 確定後の軽微な訂正は `vX.Y.Z` として公開する。
