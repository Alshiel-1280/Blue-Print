# 個人事業主向け macOS 青色申告アプリ 要件定義書

- 文書バージョン: v1.0
- 作成日: 2026-07-21
- 対象プロダクト: OSS の個人事業主向け macOS 会計・申告支援アプリ
- ステータス: 初期要件ベースライン

## 1. 文書の目的

本書は、小規模な個人事業主が macOS 上で、日常の記帳、証憑管理、請求・入金管理、決算、青色申告用データ作成までを行うオープンソースアプリケーションの要件を定義する。

本アプリは税務申告そのものを独自に送信することを目的とせず、国税庁の e-Tax WEB 版へ申告データを引き渡し、利用者が最終確認・送信する運用を基本とする。

本書は以下の判断基準として使用する。

- 実装範囲と対象外範囲の合意
- バージョンごとの実装内容の決定
- 機能・非機能テストの受入基準
- 税制改正や e-Tax 仕様変更への対応判断
- OSS コントリビューター間の設計判断の共有

## 2. プロダクトの背景と課題

既存のクラウド会計サービスは、記帳から申告までを一体化できる一方、小規模な個人事業主にとって毎年の利用料金が継続的な負担となる。また、事業・税務データを外部サービスへ預け続けること、サービス終了や料金改定に依存することも課題となる。

本アプリは、次の価値を提供する。

1. Mac だけで日常の会計処理を完結できる。
2. データを利用者自身が所有し、サービス契約なしで閲覧・出力できる。
3. 証憑から仕訳、請求、決算、申告準備までの二重入力を減らす。
4. 会計や税務に詳しくない利用者でも、未処理や不整合を把握できる。
5. OSS として計算・保存・移行処理を検証可能にする。

## 3. 用語と要求レベル

### 3.1 用語

| 用語 | 定義 |
| --- | --- |
| 事業者 | 本アプリを使用する個人事業主 |
| 利用者 | 原則として事業者本人1名 |
| 証憑 | 領収書、レシート、請求書、見積書、契約書、納品書等 |
| 電子取引原本 | メール、Web、クラウド等で電子的に受領・交付した元データ |
| 紙スキャン | 紙で受領した証憑を画像・PDF化したデータ |
| 仕訳 | 複式簿記に基づく借方・貸方の記録 |
| 転記済み | 帳簿に確定反映された状態 |
| 申告ワークスペース | 事業所得と事業外所得を申告年度単位で集約する領域 |
| 年度ルール | 税率、控除、帳票、丸め、e-Tax スキーマ等の年度別定義 |
| 正本 | 会計・証憑データの最終的な保存元。初期構成では Mac を指す |

### 3.2 要求レベル

| レベル | 意味 |
| --- | --- |
| MUST | 初回安定版 v1 に必須 |
| SHOULD | v1 系の非破壊的アップデートで対応すべき |
| FUTURE | 将来の拡張として設計上考慮する |
| OUT | 現在の対象外 |

## 4. バージョニング方針

本プロジェクトでは、利用者が変更規模と互換性を判断できるよう、以下の規則でバージョンを付与する。

### 4.1 バージョン形式

| 形式 | 用途 | 例 |
| --- | --- | --- |
| `vX` | アプリ全体の刷新、主要ワークフローの再設計、互換性に影響する変更 | `v1`、`v2` |
| `vX.Y` | 追加機能、既存機能の拡張、原則として非破壊的な変更 | `v1.1` |
| `vX.Y.Z` | バグ修正、軽微な数値・表記変更、同一年度ルールの訂正 | `v1.1.1` |

### 4.2 運用規則

- `v0.Y` は開発版とし、正式な移行保証を開始する前のマイルストーンを表す。
- `v1` を最初の一般利用可能な安定版とする。
- `vX.Y` では既存データを失わず、自動移行できなければならない。
- `vX.Y.Z` では原則として破壊的な DB スキーマ変更を行わない。
- 税額計算の誤りを修正するパッチは、影響年度、影響機能、再計算の要否をリリースノートへ明記する。
- 新しい申告年度の帳票・税制追加は、機能追加を伴うため原則 `vX.Y` とする。
- 互換性のないデータ形式変更や主要 UI の全面刷新は `vX` とする。
- すべての移行前に自動バックアップを作成し、失敗時は旧バージョンのデータを保持する。

## 5. 対象利用者と利用条件

### 5.1 主対象

- 日本国内で事業所得を得る個人事業主
- 基本的に事業者本人が1人で利用する
- 従業員および青色事業専従者を雇用していない
- 外注先への発注・支払いが発生する
- Mac を日常の会計処理に使用する
- 青色申告を行う、または青色申告を予定している
- 給与、不動産、株式・配当等の事業外所得を持つ可能性がある

### 5.2 前提条件

- 会計年度は個人事業主の暦年を基本とする。
- 通貨は日本円を基本とする。
- Mac 上のデータを正本とする。
- インターネット接続なしでも記帳・閲覧・決算ができる。
- e-Tax WEB 版の利用、マイナンバーカードによる認証、最終送信は利用者が行う。

## 6. スコープ

### 6.1 対象範囲

- 事業者・年度の初期設定
- 複式簿記による記帳
- 現金、預金、クレジットカード、売掛、買掛・未払の管理
- 銀行・カード CSV インポートと消込
- 証憑の取込、OCR、検索、仕訳との関連付け
- 請求書の作成、PDF 出力、売掛・入金管理
- 外注請求書、未払、支払い管理
- 固定資産、減価償却、家事按分、棚卸、期末処理
- 青色申告決算書の作成支援
- 給与、不動産、株式・配当、雑所得等の申告情報管理
- e-Tax WEB 版へ渡す申告データの生成
- 弥生等からの CSV 移行
- バックアップ、復元、オープン形式での全データ出力
- 将来の撮影専用 iPhone アプリを受け入れられるデータ構造

### 6.2 対象外

- 法人会計・法人税申告
- 給与計算、勤怠、年末調整
- 従業員・専従者の労務管理
- 複数ユーザー、承認ワークフロー、権限管理
- e-Tax へのアプリ内直接送信
- 銀行 API の常時自動同期
- 税理士による顧客一括管理
- 株価、ポートフォリオ、取引ごとの投資管理
- POS、在庫・製造原価、店舗別管理などの大規模業務機能
- 外貨建取引の高度な為替差損益管理
- iPhone 上での仕訳、決算、申告書編集

## 7. プロダクト原則

1. **Mac が正本**: すべての確定処理は Mac で行う。
2. **ローカルファースト**: 会計機能はクラウド契約なしで動作する。
3. **人が確定する**: OCR や自動仕訳は候補を提示し、無確認で転記しない。
4. **原本を残す**: 証憑の原本データを上書きしない。
5. **追跡可能にする**: 帳簿、仕訳、証憑、請求、支払いの関連を辿れるようにする。
6. **税制を年度分離する**: 税率・帳票・計算ロジックをアプリ本体へ固定しない。
7. **利用者を閉じ込めない**: 全データを文書化された形式で出力できるようにする。
8. **申告前に不整合を見せる**: 完了していない作業を隠さず、チェックリスト化する。

## 8. 主要ユーザーフロー

### 8.1 初回利用

1. 事業者情報と申告年度を設定する。
2. 青色申告、消費税、インボイス登録状況を設定する。
3. 標準勘定科目を生成し、必要に応じて編集する。
4. 開始残高または他社会計ソフトのデータを取り込む。
5. バックアップ先を設定する。

### 8.2 日常記帳

1. 証憑、銀行明細、カード明細、請求書のいずれかを取り込む。
2. アプリが取引日、相手先、金額、税区分、勘定科目候補を提示する。
3. 利用者が候補を確認・修正する。
4. 仕訳として確定する。
5. 月次チェックで未処理、重複、残高差異を解消する。

### 8.3 請求と入金

1. 顧客と請求内容を入力する。
2. 請求書を確定し、PDF を保存・交付する。
3. 売掛金と売上の仕訳を生成する。
4. 入金明細と請求書を消し込む。
5. 振込手数料、源泉徴収、差額がある場合は補助仕訳を生成する。

### 8.4 外注と支払い

1. 外注先と受領請求書を登録する。
2. 支払予定、税区分、インボイス区分を確認する。
3. 外注費と未払金の仕訳を生成する。
4. 支払明細と未払金を消し込む。
5. 必要な場合だけ源泉徴収・支払調書用情報を集計する。

### 8.5 年次決算と申告準備

1. 未処理証憑、未消込、残高不一致を解消する。
2. 棚卸、家事按分、減価償却、経過勘定を処理する。
3. 損益計算書と貸借対照表を確認する。
4. 給与、不動産、株式・配当等の事業外所得を登録する。
5. 青色申告決算書と申告ワークスペースを検証する。
6. e-Tax 用データを出力し、e-Tax WEB 版で最終確認・送信する。
7. 受付結果を保存し、年度をロックする。

### 10.2 データ不変条件

- 転記済み仕訳は借方合計と貸方合計が一致する。
- 転記済み仕訳は物理削除しない。
- 証憑原本は同一 ID のまま上書きしない。
- すべての証憑は原本ハッシュを持つ。
- 仕訳の税率とインボイス区分は独立して保持する。
- 年度ロック後は業務データを変更できない。
- 取消・訂正は元記録と相互参照できる。
- 税額計算結果は使用した年度ルールの ID と版を持つ。
- e-Tax 出力は使用したアプリ、スキーマ、年度ルールの版を持つ。
- インポートはバッチ単位で追跡・取消できる。

## 12. 受入条件

v1 の受入には、最低限以下を満たす必要がある。

1. 標準的な1年分の取引を入力し、仕訳帳、総勘定元帳、試算表、損益計算書、貸借対照表の金額が一致する。
2. 転記済み仕訳を削除できず、取消・訂正履歴を追跡できる。
3. 領収書を取り込み、OCR 候補を確認して仕訳へ関連付けられる。
4. 電子取引原本と紙スキャンを区別して検索・一括出力できる。
5. 適格請求書と免税・未登録事業者の10%取引を区別して集計できる。
6. 請求書発行から売掛計上、入金消込までを二重計上なく完了できる。
7. 外注請求書から未払計上、支払消込までを完了できる。
8. 固定資産、家事按分、棚卸を含む決算整理を行える。
9. 事業所得と不動産所得を分離し、給与・株式・配当の申告用情報を年度に集約できる。
10. 対象年度の青色申告決算書データと `.xtx` を生成し、e-Tax WEB 版へ読み込める。
11. 弥生のサンプル CSV をエラー明細付きで移行できる。
12. バックアップから復元後、仕訳件数、残高、証憑ハッシュが一致する。
13. 通常の会計・OCR 操作で税務データを外部へ送信しない。
14. 公式ビルド、データ形式、税制根拠、既知の制限が文書化されている。

## 13. 法令・外部仕様への対応方針

### 13.1 基本方針

- 本アプリは税務判断そのものを代替せず、計算根拠、選択肢、未対応範囲を明示する。
- 法令、国税庁の手引き、e-Tax 仕様を一次情報として年度ルールを更新する。
- 新年度対応では旧年度の計算結果が変わっていないことを回帰テストする。
- 仕様未確定の年度は「暫定対応」と表示し、確定仕様公開後に検証する。
- 法令上の判断が分かれる項目は自動確定せず、利用者の選択または e-Tax WEB 版での確認を求める。

## 14. リスクと対応

| リスク | 影響 | 対応 |
| --- | --- | --- |
| 税制・帳票が毎年変わる | 誤計算、出力不能 | 年度ルール分離、根拠付きゴールデンテスト、更新手順 |
| e-Tax 仕様が複雑 | `.xtx` が受理されない | 専用モジュール化、年度別スキーマ検証、早期試験 |
| OCR の誤読 | 誤仕訳 | 確認待ち必須、信頼度表示、原本との並列確認 |
| 証憑・DB の破損 | 保存義務を満たせない | ハッシュ、世代バックアップ、復元テスト、診断 |
| 弥生 CSV の差異 | 移行失敗 | 列マッピング、プレビュー、エラー隔離、バッチ取消 |
| 外注の源泉徴収判断 | 誤った源泉処理 | 既定オフ、説明表示、利用者選択、特殊ケース警告 |
| 株式・不動産の範囲拡大 | MVP 肥大化 | 年間報告書・専用帳簿に限定し、投資管理等は対象外 |
| OSS の税務ロジック責任 | 利用者の過信 | 対象年度・根拠・既知の制限・検証状況を明示 |
| macOS 更新 | 動作不能 | 最新＋過去2メジャーの CI・リリース検証 |

### パッチリリースの扱い

`vX.Y.Z` はロードマップ上で事前に機能を割り当てず、次の場合に随時公開する。

- クラッシュ、データ破損、重複転記等の修正
- 税額・端数・帳票マッピングの訂正
- e-Tax の同一年度仕様に対する軽微な追随
- UI 文言、ヘルプ、誤字、レイアウトの修正
- 性能・メモリ使用量の小規模改善
- 依存ライブラリの互換性・脆弱性対応

税額または申告出力へ影響するパッチは、影響を受ける年度・条件・再出力手順を必ず明記する。
