# v0.8 検証記録

## 実装範囲

- 弥生デスクトップ／オンラインの25列仕訳、弥生会計 Next の25／27列仕訳と6列期首残高をプレビュー
- 2000／2111の単一仕訳、2110／2100／2101の複合仕訳、製品別の20／100行上限を判定
- 勘定科目マッピング、補助科目の自動作成、税率・税込区分・適格／区分記載・控除割合の分離
- 正常行、貸借不一致、未対応フラグ、未対応税区分の隔離と、バッチ単位の原子的取込
- 全SQLiteテーブルのタグ付きJSONとUTF-8 CSV、証憑原本・索引、検証用SQLiteを含む持ち出し形式
- 100,000回反復SHA-256鍵導出とAES-256-GCMによる認証付き暗号化
- キーチェーンへ保存したパスフレーズを使う日次・7世代バックアップ
- 復元前の形式互換性・残高・件数確認、次回起動時復元、復元前データのロールバックコピー
- SQLite integrity_check、借貸合計、証憑原本の欠落・SHA-256不一致診断

## 公式仕様の確認

2026-07-21に弥生公式サポートを確認した。

- [弥生会計 オンライン: インポートデータの記述形式](https://support.yayoi-kk.co.jp/subcontents.html?page_id=27184): 識別フラグ、20行までの複合仕訳、日付、科目、税区分、税込金額
- [弥生会計 Next: インポートデータの記述形式](https://support.yayoi-kk.co.jp/subcontents.html?page_id=29611): 25／27列、100行までの複合仕訳、6列の期首残高、科目名上限
- [弥生会計 Next: 税区分](https://support.yayoi-kk.co.jp/subcontents.html?page_id=29612): 10%／軽減8%、内外税、適格、区分記載、100／80／70／50／30%と控除不可
- [弥生会計: 仕訳データのエクスポート](https://support.yayoi-kk.co.jp/faq_Subcontents.html?page_id=18042): 弥生インポート形式と本則課税時の出力差
